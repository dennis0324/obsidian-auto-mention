# 링크가 헤더 섹션 안에 있으면 헤더까지 포함

**날짜:** 2026-05-08  
**관련:** 사용자 요청 — “mention link에 추가할 때 해당 링크가 헤더 안에 존재하면 헤더까지 같이 넣어줘”

## 배경 / 목적

대상 노트의 `mention links`에 소스 노트를 추가할 때, 링크가 특정 헤더 아래에서 발생한 경우 그 위치로 바로 점프할 수 있도록 헤더 앵커까지 함께 저장합니다.

## 변경 요약

- 소스 본문에서 링크가 최초로 등장한 위치의 “현재 헤더”를 감지해, 대상 노트의 `mention links` 엔트리를 `[[Source#Heading]]` 형태로 기록합니다.
- 헤더가 없는 구간(문서 상단 등)에서 발생한 링크는 기존처럼 `[[Source]]`로 기록합니다.
- 기존에 이미 `mention links`에 기록된 엔트리가 있더라도, 소스 노트에서 링크의 “최초 등장 헤더”가 바뀌면 대상 노트의 엔트리도 새 `#Heading`으로 갱신됩니다. (소스 1건당 1엔트리 유지)

## 구현 메모

- 링크 스캔 시 코드 펜스( ``` ) 내부는 기존과 동일하게 무시합니다.
- 같은 소스→타겟 링크가 여러 헤더에서 반복 등장하는 경우, **첫 등장 위치**의 헤더를 사용합니다. (YAML 리스트가 소스 1건당 1엔트리 형태인 기존 구조를 유지)

## 변경된 경로 (참고)

- [ ] 소스 노트에서 링크의 최초 등장 위치를 다른 헤더 섹션으로 옮기거나(또는 헤더명을 바꿔 링크가 업데이트되도록) 한 뒤, 타겟 노트의 `mention links` 엔트리가 새 `#Heading`으로 업데이트되는지 확인
